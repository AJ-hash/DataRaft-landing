What is DataRaft?

DataRaft is a mixed-language federated query engine powered by a semantic metadata catalog. It allows you to write queries in either SQL or natural language, and seamlessly retrieves, joins, and resolves data across structured and semi-structured sources, including cloud warehouses, object stores, and APIs.

More than a query interface, DataRaft is a data marketplace. Organizations can publish and monetize datasets and pipelines while consumers can discover, subscribe to, and federate across external data assets with full semantic alignment. DataRaft not only understands your intent — it safeguards it across schema changes, naming inconsistencies, and organizational silos.

For Data Engineers

DataRaft makes your pipelines resilient to schema drift by allowing semantic-level abstraction. Instead of tying queries and jobs to brittle field names, DataRaft lets you define intent.

Basic Usage (Schema-Aware ETL)

etl_basic.py

			from dataraft import DataRaftClient
from sqlalchemy import create_engine, inspect, text
import pandas as pd

# --- Connect to DataRaft and Warehouse ---
client = DataRaftClient(jdbc_url="jdbc:dataraft://localhost:5440", auth_token="xyz")
engine = create_engine("snowflake://USER:PASSWORD@ACCOUNT/DB/SCHEMA?warehouse=ETL_WH")
inspector = inspect(engine)

# --- NL Query ---
query = "What is the average delivery time for priority orders shipped to customers in California?"
df_new = client.query(query).to_pandas()
target_table = "priority_order_metrics"

# --- Basic Schema Comparison (Exact Match) ---
if target_table in inspector.get_table_names():
    with engine.connect() as conn:
        existing_cols = conn.execute(text(f"SELECT * FROM {target_table} LIMIT 0")).keys()

    new_cols = df_new.columns.tolist()
    if set(existing_cols) == set(new_cols):
        print("Exact schema match. Appending.")
        df_new.to_sql(target_table, con=engine, if_exists="append", index=False)
    else:
        version = 2
        while f"{target_table}_v{version}" in inspector.get_table_names():
            version += 1
        new_table = f"{target_table}_v{version}"
        old_df = pd.read_sql_table(target_table, con=engine)

        # Align schemas (insert NaNs where needed)
        for col in set(new_cols) - set(old_df.columns):
            old_df[col] = None
        for col in set(old_df.columns) - set(new_cols):
            df_new[col] = None

        combined = pd.concat([old_df[df_new.columns], df_new], ignore_index=True)
        combined.to_sql(new_table, con=engine, if_exists="replace", index=False)
        print(f"Migrated to new table: {new_table}")
else:
    print("Creating new table.")
    df_new.to_sql(target_table, con=engine, if_exists="replace", index=False)

Train a Model from Production Data

train_model.py

from dataraft import DataRaftClient
from sklearn.ensemble import RandomForestClassifier

# Initialize DataRaft client
client = DataRaftClient(jdbc_url="...", auth_token="...")

# Query training data using natural language
df = client.query("Show me customer purchase events in Q1 2024 with labels for churn").to_pandas()

# Train model
features = df.drop(columns=["churn_label"])
model = RandomForestClassifier()
model.fit(features, df["churn_label"])
